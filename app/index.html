Moat
README Repository

Pre-execution firewall for smart contracts

Moat is an open-source project aimed at gating critical contract calls: simulate state before execution, evaluate team-defined rules, and enforce the outcome on-chain. It is meant to complement audits and other tooling, not replace them.

Context

Many incident responses still depend on reacting after suspicious transactions appear— for example racing the mempool or pausing an entire protocol. Those approaches are brittle, and losses in DeFi remain large; see the Rekt leaderboard for a running index of documented cases.

Moat focuses on delaying or blocking specific calls when an off-chain check fails, instead of only acting once state has already changed. Smart contract risk categories are summarized in the OWASP Smart Contract Top 10 (2026).

Some commercial guardrails are closed or scheduled to wind down; Moat explores an open, auditable path built from existing primitives.

How it works

High-level transaction path: initiation, off-chain validation, on-chain attestation, execute or timelock.

  • On-chain: a modifier on functions you choose (for example withdraw or deposit), using a Forta Firewall–style proxy pattern for attestation checks.
  • Off-chain: operators run emulation (state before and after the call), apply your rules (limits, ratios, frequency, and similar), and sign attestations. The design targets an actively validated service on EigenLayer / Symbiotic.
  • Rules: defined by the protocol team as plain invariants—what “normal” means for your system.
  • Fallback: when a call is held or rejected, a configurable timelock (for example 30 minutes) is intended to give responders time while still allowing the caller to proceed afterward, so execution is not permanently blocked by the firewall alone.

Roadmap

  • Milestone 1 — On-chain contracts, timelock, off-chain emulation backend, basic rule engine, and security review before a public release.
  • Milestone 2 — Decentralized operator set (AVS), integrations with security and data providers where useful, and starter rule templates.
  • Milestone 3 — Preset rule patterns for common DeFi surfaces so teams can start from templates and adapt.
